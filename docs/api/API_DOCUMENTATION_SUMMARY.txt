================================================================================
PARAMETER PATTERNS IDENTIFIED
================================================================================

Pagination:
- page_no (1-indexed, not 0-indexed)
- limit (default: 50)
- Response includes total_count

Date Handling:
- Format: YYYY-MM-DD (ISO format)
- Fields: from_date, to_date, til_date (endpoint dependent)
- Conversion helper: formatDate(dateValue) => string

Filtering:
- Search: q (full-text search)
- Status: specific enum values (INCOMING, PENDING, DRAFT, COMPLETED)
- Multi-select: JSON stringified arrays
- Type filters: 'animals' vs 'species'

Form Data (Multipart):
- Nested array indexing: organs[0][organ_id]
- Multiple file append: attachments
- Organ details with images

Response Structures:
- Pattern 1: {success: true, data: {result: [], total_count: number}}
- Pattern 2: {success: true, data: {list: []}}
- Pattern 3: {success: true, data: []}}
- Error Pattern: {success: false, message: string}

================================================================================
KEY DISCOVERIES
================================================================================

1. FILTER CONSTRUCTION
   - Multi-select filters are JSON stringified
   - Example: JSON.stringify(['Male', 'Female'])
   - Single values passed as strings directly

2. DATE FIELDS INCONSISTENCY
   - Animal-wise list uses: from_date, to_date
   - Species-wise list uses: from_date, til_date
   - Medical endpoints use: from_date, to_date
   - Always use YYYY-MM-DD format

3. PAGINATION MISMATCH
   - API: 1-indexed pages (page_no starts at 1)
   - MUI DataGrid: 0-indexed (starts at 0)
   - Conversion required: apiPage = gridPage + 1

4. FILE UPLOAD PATTERN
   - Uses multipart/form-data
   - Array items indexed: organs[0], organs[1], etc.
   - Sub-properties: organs[0][organ_id], organs[0][findings]
   - Multiple files: append each to FormData

5. MASTERS DATA
   - Static options retrieved on-demand
   - Manner of Death, Carcass Disposition, Units
   - Response structure: Array<{id, string_id, name, label, value}>

6. CENTER MANAGEMENT
   - Status parameter determines operation: 'add', 'update', 'active', 'inactive'
   - Path: /v1/entity/types/{status}/{necropsyId}
   - necropsyId only required for 'update' status
